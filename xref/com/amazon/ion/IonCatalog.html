16  package com.amazon.ion;
17  
18  import com.amazon.ion.system.IonSystemBuilder;
19  import com.amazon.ion.system.SimpleCatalog;
20  
21  
22  /**
23   * Collects shared symbol tables for use by an {@link IonSystem}.
24   * <p>
25   * It is expected that many applications will implement this interface to
26   * customize behavior beyond that provided by the default {@link SimpleCatalog}.
27   * A typical implementation would retrieve symbol tables from some external
28   * source.
29   * <p>
30   * To utilize a custom catalog, it must be given to the
31   * {@link IonSystemBuilder} before a system is built, or to
32   * selected methods of the {@link IonSystem} for localized use.
33   *
34   * <h2>Notes for Implementors</h2>
35   * This interface defines two methods with subtly different semantics.
36   * The first variant takes only a symbol table name, and returns the highest
37   * version possible. The second takes a version number and attempts to match it
38   * exactly, and if that's not possible it falls back to the the "best match"
39   * possible:
40   * <ul>
41   *   <li>If any versions <em>larger</em> than the requested version are
42   *     available, select the smallest among them (still larger than requested).
43   *   <li>Otherwise all available versions are <em>smaller</em> than the
44   *     requested version, so return the largest of them.
45   * </ul>
46   * <p>
47   * Catalog implementations should <em>never</em> accept substitute symbol
48   * tables and <em>never</em> return them. Substitute tables are used when the
49   * catalog cannot find an exact match, that is, the catalog cannot find an
50   * imported shared symtab with the same name, version and max_id. Refer to
51   * {@link SymbolTable}.
52   * <p>
53   * This interface is the <em>only</em> abstraction point for caching shared
54   * symbol tables. Within this library, there is no caching mechanism in place on
55   * shared symbol tables that are loaded into the {@link IonSystem}.
56   * This means that when a shared symbol table needs to be retrieved by the
57   * library's code-paths, methods of this interface are invoked directly,
58   * without any additional caching whatsoever.
59   * As such, implementors of this interface should implement their own caching
60   * mechanism if desired.
61   * <p>
62   * When encoding Ion binary data, its always best to use an exact match to the
63   * requested version whenever possible.  Earlier versions are very likely to be
64   * missing symbols that are needed by the data.  Later versions of the table
65   * could have the same problem, but that's less likely under best practices.
66   * <p>
67   * While "get latest version" is generally okay for encoding, it's not
68   * universally acceptable: one can imagine a client/server protocol where the
69   * client declares what symtab/versions it can handle, and the server needs to
70   * meet those requirements.
71   * <p>
72   * Binary <em>decoding</em> prefers an exact match, and in a couple edge cases,
73   * requires it. Therefore a single "get latest version" method is insufficient.
74   * See the
75   * <a href="http://amzn.github.io/ion-docs/docs/symbols.html">Ion Symbols page</a>
76   * for more details on this topic.
77   * <p>
78   * It's expected that many if not most applications will implement a dynamic
79   * catalog that can fetch symtabs from some source.  In such cases the catalog
80   * should make its best effort to find an exact match, and if that's not
81   * possible fall back to the best match it can acquire.
82   */
83  public interface IonCatalog
84  {
85      /**
86       * Gets a symbol table with a specific name and the highest version
87       * possible.
88       *
89       * @param name identifies the desired symbol table.
90       * @return a shared symbol table with the given name, or
91       * {@code null} if this catalog has no table with the name.
92       */
93      public SymbolTable getTable(String name);
94  
95  
96      /**
97       * Gets a desired symbol table from this catalog, using an exact match if
98       * possible.
99       * <p>
100      * Implentations must make a best effort to find an exact match.
101      * If an exact match cannot be found, then this method must make a best
102      * effort to find the best match available.
103      *
104      * @return the shared symbol table with the given name and version, when an
105      * exact match is possible. Otherwise, returns the lowest possible version
106      * larger than requested.  Otherwise, return the largest possible version
107      * lower than requested.  If no table with the name can be found, then
108      * this method returns {@code null}.
109      */
110     public SymbolTable getTable(String name, int version);
111 }
